# Задачи за упражнение преди изпит

## Задача 1. Списък с инструкции за пазаруване

Да се реализира система за управление на количества от различни видове артикули и за изпълнение на последователни инструкции върху списък за пазаруване.

### `Inventory<T>`

Реализирайте шаблонен клас `Inventory<T>`, който пази различни елементи от тип `T` и наличното количество от всеки елемент.

Един елемент трябва да присъства логически само веднъж, независимо от количеството му. Два елемента се считат за еднакви според `operator==`, дефиниран от типа `T`.

Инвентарът трябва да може да се използва с различни типове:

```cpp
Inventory<Product> products;
Inventory<Book> books;
Inventory<GameItem> backpack;
```

Приема се, че типът `T` може да бъде копиран, присвояван и сравняван за равенство.

Инвентарът трябва да позволява:

- добавяне на положително количество от даден артикул
- премахване на положително количество
- проверка дали артикул присъства
- получаване на броя различни артикули
- получаване на общия брой бройки
- проверка дали инвентарът е празен

Ако след премахване количеството стане `0`, артикулът вече не трябва да присъства в инвентара.

При опит за премахване на липсващ артикул или на повече бройки от наличните трябва да се хвърли подходящо изключение. Количество `0` не е валидно за добавяне или премахване.

### Достъп до количество чрез `[]`

Операторът `[]` трябва да приема артикул и да връща неговото текущо количество. За липсващ артикул резултатът е `0`.

```cpp
Book dune("Dune", "Frank Herbert");
Book foundation("Foundation", "Isaac Asimov");

Inventory<Book> library;

library.add(dune, 4);

std::cout << library[dune];        // 4
std::cout << library[foundation];  // 0
```

### Оператори върху инвентара

Инвентарът трябва да поддържа следното естествено поведение:

```cpp
inventory += item;  // добавя една бройка
inventory -= item;  // премахва една бройка
```

Два инвентара трябва да могат да се обединят чрез `+`. В резултата присъстват всички елементи, а количествата на общите елементи се сумират. Оригиналните обекти не се променят.

```cpp
Inventory<Product> combined = first + second;
```

Префиксните и постфиксните версии на `++` увеличават с една бройка количеството на всеки вече наличен елемент.

Префиксните и постфиксните версии на `--` намаляват с една бройка количеството на всеки елемент. Елементите, достигнали количество `0`, се премахват.

Постфиксните оператори трябва да връщат състоянието преди промяната.

```cpp
Inventory<Product> previous = inventory++;
--inventory;
```

### `Product`

Реализирайте обикновен клас `Product`, описващ продукт (артикул) в магазин.

Всеки продукт има:
- име
- единична цена
- категория (Food, Drink, Household, PersonalCare, Other, ...)

Цената не може да бъде отрицателна.

Два продукта се считат за един и същ продукт, когато имат еднакви име, цена и категория. Осигурете подходящ достъп само за четене до характеристиките на продукта.

### `Cart`

Реализирайте клас `Cart`, който наследява `Inventory<Product>` и представлява пазарна количка.

Освен всички операции на инвентара, количката трябва да може да:

- намира общата стойност на всички продукти, взимайки предвид техните количества
- намира общата стойност само на продуктите от подадена категория

Пример:

```cpp
Cart cart;

cart.add(milk, 2);
cart.add(bread, 3);

std::cout << cart[milk];  // 2
std::cout << cart.getTotalPrice();
std::cout << cart.getTotalPriceByCategory(ProductCategory::Food);
```

При празна количка общата стойност е `0`.

###  Инструкции за пазаруване

Реализирайте абстрактен клас `ShoppingInstruction`, който описва операция върху `Cart`.

Например:
```cpp
instruction->applyTo(cart);
```

Реализирайте следните видове инструкции:

#### Инструкция за добавяне на количество от продукт

- Пази продукт и положително количество. При изпълнение добавя посоченото количество.

#### Инструкция за премахване на количество продукт

- Пази продукт и положително количество. При изпълнение премахва посоченото количество.

#### Инструкция за допълване до минимално желано количество

- Пази продукт и минимално желано количество.
- След изпълнение количеството на продукта трябва да бъде поне равно на зададения минимум. Ако то вече е достатъчно, не се извършва промяна.

#### Инструкция за сливане с друг списък

- Пази друга количка и при изпълнение добавя нейното съдържание към текущата.

### Инструкция за смяна на продукт

- Пази стар и нов продукт. 
- При изпълнение всички бройки от стария продукт се заменят със същия брой бройки от новия. Ако старият продукт липсва, се хвърля подходящо изключение. Ако двата продукта са равни, не се извършва промяна.

### `ShoppingPlan`

Реализирайте клас `ShoppingPlan`, който управлява:

- текуща количка
- колекция от инструкции

Планът трябва да позволява:

- създаване с празна или предварително попълнена количка
- добавяне на инструкция
- премахване на инструкция по индекс
- последователно изпълнение на всички инструкции върху количката
- достъп само за четене до текущата количка
- получаване на броя инструкции

При невалидна инструкция или невалиден индекс трябва да се хвърли подходящо изключение.

Ако дадена инструкция хвърли изключение:

- всички промени, извършени от нея, трябва да бъдат отменени и количката да остане в състоянието си отпреди изпълнението на инструкцията
- изпълнението продължава със следващата инструкция
- промените от предишните успешно изпълнени инструкции се запазват
- трябва да предадена информация за неуспешната инструкция и причината за неуспеха, достъпна след приключване на изпълнението

### Примерна употреба

```cpp
Product milk(
    "Milk",
    2.80,
    ProductCategory::Drink
);

Product bread(
    "Bread",
    2.20,
    ProductCategory::Food
);

Product eggs(
    "Eggs",
    4.50,
    ProductCategory::Food
);

Cart initialCart;
initialCart.add(milk, 1);
initialCart.add(bread, 2);

ShoppingPlan plan(initialCart);

plan.addInstruction(/* добави 2 опаковки яйца */);
plan.addInstruction(/* премахни 1 хляб */);
plan.addInstruction(/* осигури поне 4 млека */);

plan.execute();

const Cart& result = plan.getCart();

std::cout << result[milk];
std::cout << result[bread];
std::cout << result[eggs];
```

## Задача 2. Частично дефинирани функции и композиция на функции

Да се реализира програма на C++ за представяне и композиране на реални функции върху една променлива (реално число).

Всички функции приемат аргумент от тип `double` и като резултат връщат `std::optional<double>`, който има стойност, ако функцията е дефинирана за подадената стойност.

Някои функции са дефинирани за всяко реално число, а други могат да бъдат недефинирани за определени стойности.

### `Function`

Реализирайте абстрактен клас `Function`.

Всеки обект от йерархията трябва да може да бъде извикан чрез синтаксис, подобен на обикновена функция:

```cpp
std::optional<double> result = func(3.5);
```

Резултатът съдържа изчислената стойност, само ако функцията е дефинирана за подадената стойност.

### `LinearFunction` - линейна функция

Представя функцията:

```text
f(x) = ax + b
```

Тя е дефинирана за всяко реално число.

Пример:

```cpp
LinearFunction f1(2, 1);

auto result = f1(3);
// result съдържа 7
```

### `QuadraticFunction` - квадратна функция

Представя функцията:

```text
f(x) = ax² + bx + c
```

Тя е дефинирана за всяко реално число.

```cpp
QuadraticFunction f2(1, 0, 0);

auto result = f2(4);
// result съдържа 16
```

### `IntervalRestrictedFunction`

Обхваща друга функция, ограничено до затворен интервал `[left, right]`.

Извън интервала функцията е недефинирана. Вътре в интервала се използва поведението на вложената функция.

При невалидно зададен интервал трябва да се хвърли подходящо изключение.

```cpp
QuadraticFunction func(1, 0, 0);

IntervalRestrictedFunction restricted(func, -2, 2);

restricted(2);  // 4
restricted(3);  // empty
```

### `PuncturedFunction`

Обхваща друга функция, която допълнително е недефинирана в зададен набор от точки.

```cpp
LinearFunction linearFunc(2, 1);

PuncturedFunction punctured(
    linearFunc, {1, 3, 5}
);

punctured(2);  // 5
punctured(3);  // empty
```

### `PiecewiseFunction`

Представя функция, дефинирана чрез две други функции и гранична точка p.

Нека `leftFunction` и `rightFunction` са две функции. Тогава:

```
f(x) = leftFunction(x),  ако x < p
f(x) е недефинирана,    ако x = p
f(x) = rightFunction(x), ако x > p
```
Функциите от двете страни на границата могат да бъдат от произволен конкретен вид, наследяващ `Function`.

`PiecewiseFunction` трябва да съхранява независими полиморфни копия на двете подадени функции. Последващи промени или унищожаване на оригиналните обекти не трябва да влияят върху нея.

Пример:
```cpp
LinearFunction left(1, 0);
// left(x) = x

QuadraticFunction right(1, 0, 0);
// right(x) = x²

PiecewiseFunction piecewise(
    left,
    right,
    3
);

piecewise(2);  // left(2) = 2
piecewise(3);  // undefined at this point
piecewise(4);  // right(4) = 16
```

Отделните функции също могат да бъдат частично дефинирани:

```cpp
LinearFunction linear(2, 1);

PuncturedFunction punctured(
    linear,
    {1, 2}
);

QuadraticFunction quadratic(1, 0, 0);

PiecewiseFunction another(
    punctured,
    quadratic,
    5
);

another(2);  // std::nullopt
another(4);  // punctured(4) = 9
another(5);  // std::nullopt
another(6);  // quadratic(6) = 36
```

### `ComposedFunction` - композиция на функции

Реализирайте клас `ComposedFunction`, който също е вид `Function` и представя композиция на произволен брой функции.

Функциите трябва да могат да се добавят последователно чрез `then()`:

```cpp
ComposedFunction composition;

composition
    .then(first)
    .then(second)
    .then(third);
```

Ако функциите са добавени в реда `f`, `g`, `h`, резултатът е:

```text
h(g(f(x)))
```

Тоест резултатът от всяка функция се подава като аргумент на следващата.

Ако някоя функция е недефинирана за получения междинен аргумент, то останалите композирани функции не се изчисляват и броим цялата композиция за недефинирана в тази точка.

Празната композиция трябва да се държи като тъждествената функция (идентитет):

```text
f(x) = x
```

Композицията трябва да позволява:

- добавяне на функция
- премахване на функция по индекс
- достъп до функция по индекс
- получаване на броя функции
- проверка дали е празна
- независимо копиране

Тъй като `ComposedFunction` също е функция, една композиция трябва да може да участва като елемент в друга композиция.

### Примерна употреба

```cpp
LinearFunction linear(1, 1);
// linear(x) = x + 1

LinearFunction leftBranch(0, 0);
// leftBranch(x) = 0

LinearFunction rightBranch(0, 10);
// rightBranch(x) = 10

PiecewiseFunction piecewise(
    leftBranch,
    rightBranch,
    3
);

QuadraticFunction quadratic(1, 0, 0);
// quadratic(x) = x²

ComposedFunction composition;

composition
    .then(linear)
    .then(piecewise)
    .then(quadratic);

auto first = composition(1);
// linear(1) = 2
// piecewise(2) = leftBranch(2) = 0
// quadratic(0) = 0

auto second = composition(2);
// linear(2) = 3
// piecewise(3) е недефинирана
// second -> std::nullopt

auto third = composition(4);
// linear(4) = 5
// piecewise(5) = rightBranch(5) = 10
// quadratic(10) = 100
```
